🥊 重复劳动变技能包

AI 的记忆会丢，但 skill 文件不会

1 我干成了啥

在小米汽车做 NPS（净推荐值）数据分析。一开始，每次拿到调研数据都要从零开始——重新理解需求、重新写脚本、重新调格式。AI 每次换一个对话就全忘了。

两个月后，我们把所有重复性工作打包成了可复用的 skill 系统：

行 Python 代码
15+
独立可运行脚本
4
功能模块
583
行 SKILL.md

现在 AI 接到 NPS 数据分析任务，不需要重新理解需求，直接调用对应的 skill 就能跑。

2 关键那一招

别急着抽象。先在真实任务里反复跑，攒够"手感"，再打包成技能包。

很多人一上来就想让 AI "记住"怎么做某件事。但 AI 的上下文窗口有限，换个对话就全忘了。真正的解法不是让 AI 记住，而是把经验固化成文件。

💡 核心逻辑：先用实战攒经验 → 再用脚本锁流程 → 最后用版本迭代打磨到丝滑。任何你做了 3 次以上的事，第 4 次就该变成 skill。

三步走

第一步 · 4月中旬
实战跑通，攒够"手感"
别急着抽象。先在真实任务里反复跑几轮，搞清楚哪些步骤是固定的、哪些会变、哪些坑只有踩过才知道。
第二步 · 5月18日
打包成"可复用脚本库"
当同一类任务做了 3 次以上，把核心逻辑抽出来写成脚本，按功能模块组织目录。
第三步 · 持续迭代
用"版本迭代"打磨到定稿
技能包不是一次写好的。在真实使用中反复调优，记录每一个"坑"和"确认的规则"，直到格式锁死。

3 演示 / 真实案例

Step 1：从"每次都翻车"开始

4月15日，Lu 给了我一份 62MB 的年调 rawdata PDF。我连图表数据都提取不出来。

😅 第一次尝试：直接用常规方法解析 PDF → 图表数据全部丢失，提取失败。

第二天换了 PyMuPDF + 图像识别才搞定。之后两周密集实战——4维深度分析、车身稳定性专项、因子挖掘、品牌契合度分析……每次都是从头写代码、从头查字段映射、从头调格式。

Step 2：打包成 nps-data skill

5月18日，Lu 说："把所有 NPS 数据处理能力整理成 skill。"我回看过去一个月的实战代码，拆成 4 个模块：

# nps-data skill 目录结构 scripts/ — rawdata 质检 + 汇总生成 + 交叉校验（6个脚本） qc/ — 月调/年调/新车调研的 QC 流程（6个脚本） trend/ — 趋势图表 + 品牌计算（8个脚本） factor/ — RandomForest 因子挖掘

一共 5,377 行 Python，15+ 个可独立运行的脚本。现在 AI 拿到 NPS 数据，直接调用对应模块，不用重新理解数据结构。

Step 3：nps-briefing 的版本地狱

技能包不是一次写好的。nps-briefing（NPS 简报生成器）从 v1 迭代到 v2.8.4，核心战场是飞书卡片渲染：

版本问题解法
v1 用 message 工具发卡片 → 表格渲染成代码块（飞书 bug） 换方案
v2 改用 feishu_im_message + msg_type=interactive 表格正常了 ✅
v2.5 柱状图分组不支持 改 markdown 表格 + 进度条
v2.8.2 add_md() 拆分会导致表格丢失 合并为单个元素
v2.8.4 5月26日 Lu 确认最终格式 锁死所有排版规则 🔒

最终的 nps-briefing SKILL.md = 583 行，包含 11 个区块的固定顺序和每条渲染规则。AI 每次生成简报，严格按这个文件执行，格式不再出错。

4 你们怎么用上

这招适用于所有"重复性高 + 流程固定 + 需要领域知识"的工作：

📊 数据类

月报生成、数据清洗、看板更新 → 打包成 data skill

📝 报告类

周报/季报/分析报告 → 打包成 briefing skill

🤝 协作类

审批流程、会议纪要、任务分发 → 打包成 workflow skill

✅ 判断标准：任何你做了 3 次以上的事 → 第 4 次就该变成 skill。不用一开始就完美，先跑通，再迭代。
