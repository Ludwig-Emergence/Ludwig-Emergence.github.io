实战案例：30 分钟从零到上线

🚀 一个人就是一家公司：OPC 时代

在开始之前，先聊一个正在发生的趋势：OPC（One Person Company）——一人公司。

Anthropic（Claude 的开发公司）在 2026 年 5 月发布了一份《创始人手册：打造 AI 原生创业公司》，里面有一个核心观点：

"AI 把'谁可以开公司、谁可以做产品'这件事拉回到同一条起跑线。瓶颈不再是'你能造出什么'，而是'你选择去造什么'。"
—— Anthropic《The Founder's Playbook》

过去创业需要团队：一个写代码的、一个做产品的、一个跑市场的。现在，一个人 + AI 就能覆盖所有角色。这不再是鸡汤，而是正在发生的现实。

OPC 的核心逻辑：

  • 你负责方向和判断——做什么、为什么做、为谁做
  • AI 负责执行——写代码、做设计、搞运营、写文档
  • 你的效率不再取决于你懂多少技能，而取决于你能不能把想法说清楚

💬 真实对话还原

下面是今天真实发生的对话。一个 MEM 研究生（非计算机背景）和 AI 协作，从零搭建了一个公开网站。整个过程没有手写一行代码。

这个网站——你现在正在看的这个——就是用这个流程做出来的。

第 1 分钟 · 说出目标
👤
我想做个网页挂在公网上面，你知道怎么操作吗？
🤖
推荐 GitHub Pages，免费，不需要服务器。需要三样东西：网页文件 + 托管 + 域名。你有 GitHub 账号吗？
第 3 分钟 · 确认方向
👤
没有 github 账号
🤖
那第一步就是注册。去 https://github.com 点 Sign up，用户名建议简短好记，因为会变成你的网址。
第 5 分钟 · 要求 AI 直接操作
👤
你不能直接操作 github 吗？
🤖
可以。我帮你登录 GitHub → 创建仓库 → 上传文件 → 开启 Pages。你在浏览器里点一次授权就行。

🤖 AI 自动执行了这些操作

→ 执行 gh auth login，生成授权码
→ 用户在浏览器点击一次"授权"
→ AI 自动创建仓库 Ludwig-Emergence.github.io
→ AI 自动编写 HTML + CSS 页面
→ AI 自动上传文件、开启 GitHub Pages
→ 等待 1 分钟部署完成
✅ 结果：网站上线，地址 https://Ludwig-Emergence.github.io ，全世界任何人可以访问。用户全程只做了一件事：点了一下授权。
第 10 分钟 · 迭代内容
👤
写一篇科普文章，关于AI使用的四个层级，结合今天的课程内容...
🤖
写好了。排版完成，包含四个层级详解、对比表、手把手教程、常见问题。已发布上线。
第 20 分钟 · 继续优化
👤
再加一个实战案例部分，把我今天的过程写进去。
🤖
加好了，包含对话还原、逐步拆解、复盘表格。已上线。
第 25 分钟 · 手机适配
👤
手机打开以后适配性不是很好，优化一下。
🤖
优化了：标题缩小、表格可横滑、代码块自动换行、间距收紧。已上线，刷新看效果。

📊 复盘：谁做了什么？

人类做的AI 做的
说"我想做个网站"选择技术方案
注册 GitHub 账号登录、创建仓库
浏览器点了一次授权编写 HTML/CSS 代码
描述文章内容和结构排版、上传、部署
说"再加个案例"修改代码、重新发布
说"手机适配不好"优化响应式布局
总计：约 8 句话总计：数百行代码 + 全套部署

🔑 这就是 OPC 的工作方式

回到开头的那个概念。看看这个案例里的角色分工：

创始人（你）做的事

🧠 定义目标——"我要做个网站"
🧠 做决策——"用 GitHub Pages"、"写四个层级的内容"
🧠 质量把控——"手机适配不好，改一下"
🧠 方向判断——"加个实战案例会更好"

AI（你的"员工"）做的事

⚙️ 技术选型、环境搭建
⚙️ 编写代码、设计排版
⚙️ 部署上线、域名配置
⚙️ 内容生成、迭代优化
💡 注意到了吗？创始人从头到尾没有写一行代码、没有碰终端、没有配置服务器。但一个完整的网站从无到有上线了。这就是 OPC 的核心：你不需要什么都会，你需要知道要什么。
